대시보드가 갑자기 느려졌다

Software중급10분

왜 이 링크를 받았나요?

Frontend/Full-stack 사용자가 성능 문제를 브라우저, API, 데이터 크기 관점에서 나눠보는 연습입니다.

1. 상황

운영 대시보드 첫 화면 로딩이 2초에서 9초로 늘었습니다. 최근 UI 배포와 API 변경이 모두 있었고, 일부 고객만 느리다고 합니다.

4. 답변 비교

약한 답변

프론트 번들이 커진 것 같으니 번들 사이즈를 줄이겠습니다.

괜찮은 답변

먼저 느린 구간을 네트워크/API/렌더링/데이터 크기로 나누고 특정 고객/브라우저/기간에 집중되는지 확인하겠습니다.

강한 답변

먼저 RUM과 로그로 느린 시간이 어디에 쓰이는지 분해하겠습니다. TTFB/API latency가 늘었는지, payload가 커졌는지, JS bundle/hydration/rendering이 문제인지 나눕니다. 일부 고객만 느리다면 데이터 양, 권한, 브라우저, region, feature flag를 세그먼트로 확인하고, 임시로 pagination/cache/skeleton/쿼리 최적화 중 영향도 높은 조치를 선택하겠습니다.
