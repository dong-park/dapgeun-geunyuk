4. 답변 비교

약한 답변

Lighthouse가 좋다면 프론트 문제는 아닌 것 같습니다.

괜찮은 답변

실제 사용자 RUM을 붙여 느린 디바이스와 네트워크 세그먼트를 확인하고, 큰 차트나 테이블은 lazy loading하겠습니다.

강한 답변

Lighthouse는 통제된 lab 지표라 실제 고객의 저사양 디바이스, 느린 네트워크, 큰 계정 데이터 문제를 놓칠 수 있습니다. RUM으로 LCP, INP, long task, API waterfall을 디바이스/네트워크/고객 계정 규모별로 나눠 보겠습니다. 개선은 첫 화면 핵심 정보 우선 렌더링, 차트 지연 로딩, 테이블 가상화/페이지네이션, API 응답 축소, skeleton UX로 나눕니다. 이후 p75/p95 기준 성능 budget과 배포 후 회귀 알림을 둡니다.
