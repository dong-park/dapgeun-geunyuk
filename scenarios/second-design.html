Designer / UX 두 번째 링크

Designer / UX의 두 번째 학습페이지입니다. 기능 존재 여부가 아니라 사용자의 탐색 맥락, 정보 구조, 용어 문제로 설명하는 연습입니다.

1. 실제 상황

고객센터에 “구독 해지를 못 찾겠다”는 문의가 늘었습니다. 실제로 해지 버튼은 설정 > 결제 관리 > 플랜 상세 하단에 있습니다. PM은 “화면에 있으니 문제 없다”고 말하지만, 사용자 테스트에서는 대부분 1분 안에 찾지 못했습니다.

질문: 이 문제를 어떻게 정의하고 어떤 디자인 개선을 제안하겠습니까?

4. 답변 비교

약한 답변

버튼을 더 크게 만들고 색을 진하게 하겠습니다.

괜찮은 답변

해지를 찾는 사용자의 경로를 기준으로 정보 구조를 바꾸고, 설정이나 결제 화면에서 더 명확한 라벨로 노출하겠습니다.

강한 답변

문제는 버튼이 존재하느냐가 아니라 사용자가 자신의 목표 언어로 예측 가능한 위치에서 찾을 수 있느냐입니다. 해지를 숨기는 경험은 단기 retention에는 좋아 보일 수 있지만 신뢰와 CS 비용을 해칩니다. “구독 관리” 같은 사용자의 용어로 상위 경로를 명확히 하고, 결제/플랜 화면에서 해지와 다운그레이드 옵션을 투명하게 제공합니다. 검증은 해지 경로 성공률, 탐색 시간, 관련 CS, 해지 후 재가입 의향까지 보겠습니다.

5. 면접관 꼬리질문

  1. 해지를 쉽게 만들면 churn이 늘지 않나요?
  2. 비즈니스가 원하지 않는 기능도 UX는 쉽게 만들어야 하나요?
  3. 사용자 테스트 5명 결과를 어떻게 설득하나요?
