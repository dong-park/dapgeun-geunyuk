# 답변근육 제품화 로드맵

## 제품 가설

답변근육은 단순 면접 질문 모음이 아니라, 사용자가 자신의 경력/목표/약점에 맞는 시나리오를 받고 `먼저 답변 → 구조 비교 → 꼬리질문 → 다음 링크`로 반복 훈련하는 개인 답변 코치다.

## 핵심 사용자

- 이직 준비 중인 1~7년차 실무자
- 면접에서 지식은 있는데 말이 길어지거나 구조화가 약한 사람
- PM/개발/데이터/디자인/그로스/리더십 질문을 실전 상황형으로 연습하고 싶은 사람

## 1차 제품화 목표

1. **방문자가 30초 안에 제품을 이해한다.**
   - 무엇을 받는지: 개인화 시나리오 링크
   - 어떻게 훈련하는지: 먼저 답하고, 비교하고, 꼬리질문까지 연습
   - 왜 다른지: 읽기 콘텐츠가 아니라 답변 루틴

2. **첫 방문에서 바로 한 번 답해보게 만든다.**
   - 샘플 입력
   - 추천 링크 3개
   - 이어하기/임시저장

3. **구독 전환의 최소 신호를 모은다.**
   - 프로필 제출
   - 답변 작성 시작
   - 다음 링크 요청/메일 발송 의도

## Phase 1 — 정적 MVP 고도화

- 랜딩에 `오늘의 답변 루틴` 추가
- 개인화 기준을 더 명확히 설명
- 저장된 프로필/초안 기반 `이어하기` 영역 추가
- 구독 플랜/운영 방식 설명 추가
- 각 시나리오 페이지의 CTA를 `다음 링크 요청`으로 통일

## Phase 2 — 가벼운 서버/저장

상세 구현 계획: [`docs/delivery-system-plan.md`](./delivery-system-plan.md)

- 프로필 제출 저장: 1차는 서버 내 SQLite, 이후 Supabase/Postgres로 이전 가능하게 설계
- 에이전트가 생성한 루틴과 첫 3개 학습지를 DB에 저장
- `delivery_queue` 기반으로 매일/주 3회 큐레이션 학습지 발송
- 답변 시작/해설 열기/복사 클릭 이벤트 저장
- 관리자용 신청자/큐/실패 로그 조회 API
- 메일 발송은 Resend/SMTP adapter, 메시지는 Telegram adapter로 단계적 확장

## Phase 3 — 개인화 루프

- 사용자의 약점 태그와 최근 답변을 기반으로 다음 시나리오 추천
- 답변 첨삭 프롬프트 자동 생성
- 난이도/직무/약점별 커리큘럼
- 주간 리포트: 많이 약한 답변 습관, 다음 훈련 3개

## 당장 볼 지표

- 랜딩 CTA 클릭률
- 프로필 폼 완료율
- 추천 링크 클릭률
- scenario textarea 입력 시작률
- 해설 열기 전 평균 작성 글자 수
- `첨삭 프롬프트 복사` 클릭률
- 다음 링크 요청률

## 포지셔닝 문장

> 면접 답변을 읽는 콘텐츠가 아니라, 내 경력에 맞는 질문을 받고 먼저 써보며 답변 구조를 키우는 개인 훈련 링크 구독.
